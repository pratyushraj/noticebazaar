# Creator Armour - Pre-Launch Checklist

> **Last Updated:** March 31, 2026
> **Status:** 🔴 Not Ready for Launch

This checklist must be completed before launching Creator Armour to production.

---

## 1. Backend Checklist

### API Endpoints
- [ ] All API endpoints return consistent response format
- [ ] All endpoints have proper error handling
- [ ] All endpoints have input validation (Zod schemas)
- [ ] All endpoints have rate limiting
- [ ] All endpoints log errors to Sentry
- [ ] Health check endpoint works (`/health`)
- [ ] API documentation is up to date

### Authentication & Authorization
- [ ] JWT tokens expire appropriately (access: 1hr, refresh: 7 days)
- [ ] Refresh token rotation is enabled
- [ ] OTP expiration is set (5 minutes)
- [ ] OTP rate limiting is enforced (5 per hour)
- [ ] Password reset flow works end-to-end
- [ ] Magic link flow works end-to-end
- [ ] Session invalidation works on password change

### Deal State Machine
- [ ] All state transitions are validated
- [ ] Invalid transitions return clear error messages
- [ ] Deal events are logged correctly
- [ ] Timeline shows all events in order
- [ ] Terminal states cannot be changed (except by admin)

### Contract Generation
- [ ] Contract PDF generates correctly
- [ ] Contract includes all required fields
- [ ] Contract signature flow works
- [ ] Signed contract is stored securely
- [ ] Contract can be regenerated by admin

### Invoice Generation
- [ ] Invoice PDF generates correctly
- [ ] Invoice includes GST details
- [ ] Invoice includes payment details
- [ ] Invoice numbering is sequential

### Email System
- [ ] OTP emails are sent successfully
- [ ] Contract emails are sent successfully
- [ ] Payment confirmation emails work
- [ ] Email templates are professional
- [ ] Email links work correctly
- [ ] Unsubscribe link is included (marketing emails)

## 3. Database Checklist

### Migrations
- [ ] All migrations are applied
- [ ] Migration for deal_events table is applied
- [ ] Migration for current_state column is applied
- [ ] All indexes are created
- [ ] All constraints are valid

### Row Level Security
- [ ] RLS is enabled on all tables
- [ ] RLS policies are correct
- [ ] Service role bypasses RLS correctly
- [ ] No data leakage between users

### Backups
- [ ] Automated backups are configured
- [ ] Backup retention is set (7 days minimum)
- [ ] Point-in-time recovery is enabled
- [ ] Backup restore has been tested

### Performance
- [ ] Slow queries are optimized
- [ ] Indexes are used effectively
- [ ] Connection pooling is configured
- [ ] Query timeouts are set

## 4. Security Checklist

### Input Validation
- [ ] All API inputs are validated with Zod
- [ ] File uploads are validated (type, size)
- [ ] SQL injection is prevented (parameterized queries)
- [ ] XSS is prevented (DOMPurify)
- [ ] CSRF protection is enabled

### Rate Limiting
- [ ] General API rate limit (100/15min)
- [ ] Auth endpoints rate limit (10/15min)
- [ ] OTP rate limit (5/hour)
- [ ] Contract generation rate limit (20/hour)

### Headers & CORS
- [ ] Security headers are set (Helmet)
- [ ] CORS is configured correctly
- [ ] CSP headers are set
- [ ] HSTS is enabled

### Secrets & Environment
- [ ] No secrets in code
- [ ] All secrets in environment variables
- [ ] .env files are in .gitignore
- [ ] Production secrets are rotated

### File Storage
- [ ] Storage buckets have correct permissions
- [ ] File uploads are scanned for malware
- [ ] Signed URLs expire appropriately
- [ ] File size limits are enforced
